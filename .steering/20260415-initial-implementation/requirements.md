# 要求内容（初回実装）

## 概要

`docs/product-requirements.md` で定義された v1 スコープをすべて実装する。
Next.js App Router + Tailwind CSS を用いた時計アプリの初回実装。

---

## 実装対象機能

### US-01 / US-02：現在時刻の表示

- ページを開いた瞬間に現在時刻（HH:MM:SS）を表示する
- `setInterval`（1000ms）で1秒ごとにリアルタイム更新する
- 時・分・秒はゼロパディングした2桁で表示する（例：09:05:03）
- タブが非アクティブになっても再アクティブ後1秒以内に現在時刻へ再同期する

### US-03：日付の表示

- 年・月・日・曜日を日本語（ja-JP ロケール固定）で表示する
- 表示形式：`YYYY年M月D日 曜日`（月・日はゼロ埋めなし）
- 曜日は `long` 形式（例：水曜日）
- 日付変わり目（23:59:59 → 00:00:00）で正しく切り替わること

### US-04：レスポンシブデザイン

- 320px / 375px / 768px / 1440px / 1920px の各ビューポートで横スクロールなし
- 320px 幅でコンテンツが viewport 内に収まる
- モバイル（375px）で時刻 32px 以上・日付 16px 以上

### アクセシビリティ

- `<time>` 要素を使用してセマンティクスを確保する
- `dateTime` 属性を付与する（時刻：`HH:MM:SS`、日付：`YYYY-MM-DD`）
- スクリーンリーダーの読み上げ順は時刻→日付（DOM 順に従う）
- 時刻テキストと背景のコントラスト比が 4.5:1 以上
- ブラウザのフォントサイズを 200% に拡大しても情報が欠落しない

## 受け入れ条件

`docs/product-requirements.md` §7 の全項目を満たすこと。以下に全条件を列挙する。

### 時刻表示（US-01 / US-02）

- [ ] ページ表示から 1秒以内 に時刻文字列（HH:MM:SS）が画面に表示される
- [ ] 表示時刻はシステム時刻との差分が通常時 ±1秒以内
- [ ] 時・分・秒はゼロパディングした2桁で表示される（例：09:05:03）
- [ ] タブが非アクティブになっても再アクティブ後 1秒以内 に現在時刻へ再同期される
- [ ] コンポーネントアンマウント時に `clearInterval` が呼び出されることをユニットテストで確認できる

### 日付表示（US-03）

- [ ] 日付が日本語・ja-JP ロケール固定で表示される（例：2026年4月15日 水曜日）
- [ ] 日付のフォーマットはゼロ埋めなし（例：4月5日、4月15日）
- [ ] 曜日は漢字2文字＋「曜日」で表示される（例：水曜日）
- [ ] 日付変わり目（23:59:59 → 00:00:00）で日付・曜日が正しく切り替わる

### レスポンシブ（US-04）

- [ ] 320px / 375px / 768px / 1440px / 1920px の各ビューポートで横スクロールが発生しない
- [ ] 320px 幅でコンテンツが viewport 内に収まる
- [ ] モバイル（375px）で時刻が 32px 以上、日付が 16px 以上で表示される

### アクセシビリティ

- [ ] 時刻テキストと背景のコントラスト比が 4.5:1 以上（axe で確認）
- [ ] 時刻・日付が「時刻→日付」の順でスクリーンリーダーに読み上げられる
- [ ] ブラウザのフォントサイズを 200% に拡大しても情報が欠落しない

### エラー・例外

- [ ] JavaScript が無効な環境では `<noscript>` タグにより「JavaScriptを有効にしてください」と表示される
- [ ] Next.js Hydration 差分によるエラーが発生しない（ブラウザコンソールに `hydration warning / error` が出ないことで確認）

### テスト

- [ ] ユニットテストカバレッジ 80% 以上（statement・branch・function・line）

---

## 制約事項

- v1 スコープ外は実装しない（ダークモード・タイムゾーン選択・12時間表記・アラームなど）
- 外部 API・データベースは使用しない
- ロケールは `ja-JP` 固定
- タイムゾーンはブラウザのローカルタイムゾーンに従う
- **Hydration 対策**：時刻の初期値は `null` とし、`useEffect` 内でのみクライアント側で初期化する（SSR/CSR 差分を防ぐ）
- **JS 無効対応**：`layout.tsx` の `<noscript>` タグに「JavaScriptを有効にしてください」メッセージを設置する
