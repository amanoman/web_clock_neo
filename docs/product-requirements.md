# プロダクト要求定義書

## 1. プロダクトビジョンと目的

### ビジョン
シンプルで美しく、常に正確な時刻を表示するWebアプリケーション。

### 目的
- モダンなUIフレームワーク（Next.js + Tailwind CSS）を用いた高品質な時計アプリの実現
- スペック駆動開発とハーネスエンジニアリングの組み合わせによる自律的な開発プロセスの検証

---

## 2. ターゲットユーザーと課題・ニーズ

### ターゲットユーザー
- ブラウザ上でシンプルかつ視認性の高い時計を必要とするユーザー
- デスクトップ・モバイルを問わず利用するユーザー

> **ロケール・タイムゾーン方針：** 本アプリは日本語UI固定（ja-JPロケール）を採用する。時刻はブラウザのローカルタイムゾーンに従うため、海外ブラウザでは現地時刻が日本語表記で表示される。これはv1の意図的な設計であり、将来のi18n対応はv2以降のスコープとする。

### 課題・ニーズ
- 一目で現在時刻を確認したい
- デザインがシンプルで目に優しい
- どのデバイスからでも快適に使える

---

## 3. 主要な機能一覧

| 機能 | 説明 | スコープ |
|------|------|----------|
| 現在時刻の表示 | 時・分・秒をリアルタイムで表示（24時間表記） | v1 必須 |
| 日付の表示 | 年・月・日・曜日を日本語で表示 | v1 必須 |
| レスポンシブデザイン | デスクトップ・モバイル両対応 | v1 必須 |
| ダークモード | ライト/ダーク切り替え | v2 以降（v1スコープ外） |

### v1スコープ外（明示的に除外）
以下は今回の実装対象外とする：
- ダークモード
- タイムゾーン選択・切り替え
- 12時間/24時間切り替え
- アラーム・タイマー機能
- PWA・オフライン対応
- 世界時計（複数タイムゾーン表示）
- 秒表示のON/OFF設定

## 4. 成功の定義

> **注：** 成功の定義はプロダクトとしての**理想状態**を表す。最低出荷ラインは §4-2「v1リリース基準」を参照。成功の定義をすべて満たさなくてもリリース可能な項目がある。

- 表示時刻がシステム時刻との誤差 **±1秒以内** で更新される
- 代表ビューポート（320px / 375px / 768px / 1440px / 1920px）で横スクロールが発生しない
- Lighthouseスコア 90以上（Performance・Accessibility）※測定条件は §9 参照
- ユニットテストカバレッジ 80%以上（statement / branch / function / line）

- Hydrationエラーがブラウザコンソールに出ないこと
- ユニットテストカバレッジ 80%以上

### ストレッチゴール（未達でもリリース可）
- Lighthouseスコア 90以上

## 5. ビジネス要件

- 外部APIや認証機能は不要
- サーバーサイドの処理は最小限（静的サイトに近い構成）

| ID | 優先度 | ストーリー |
|----|--------|-----------|
| US-01 | Must | ユーザーとして、ページを開いた瞬間に現在時刻を確認したい |

| US-03 | Must | ユーザーとして、今日の日付と曜日を同時に確認したい |

## 7. 受け入れ条件

### US-01 / US-02：時刻表示
- [ ] ページ表示から **1秒以内** に時刻文字列（HH:MM:SS）が画面に表示される（Lighthouseのパフォーマンス計測と同条件）
- [ ] 表示時刻はシステム時刻との差分が通常時 **±1秒以内** である
- [ ] 時・分・秒はゼロパディングした2桁で表示される（例：09:05:03）
- [ ] タブが非アクティブになっても再アクティブ後 **1秒以内** に現在時刻へ再同期される
- [ ] コンポーネントアンマウント時に `clearInterval` が呼び出されることをユニットテストで確認できる

### US-03：日付表示
- [ ] 日付が日本語・ja-JPロケール固定で表示される（例：2026年4月15日 水曜日）
- [ ] 日付のフォーマットはゼロ埋めなし（例：4月5日、4月15日）
- [ ] 曜日は漢字2文字＋「曜日」で表示される（例：水曜日）
- [ ] 日付変わり目（23:59:59 → 00:00:00）で日付・曜日が正しく切り替わる

### US-04：レスポンシブ
- [ ] 320px / 375px / 768px / 1440px / 1920px の各ビューポートで横スクロールが発生しない
- [ ] 320px幅でコンテンツが viewport 内に収まる
- [ ] モバイル（375px）で時刻が **32px以上**、日付が **16px以上** で表示される

### アクセシビリティ
- [ ] 時刻テキストと背景のコントラスト比が **4.5:1以上** である（axeで確認）
- [ ] 時刻・日付が「時刻→日付」の順でスクリーンリーダーに読み上げられる（スクリーンリーダー手動確認）
- [ ] ブラウザのフォントサイズを200%に拡大しても情報が欠落しない

### エラー・例外
- [ ] JavaScript が無効な環境では `<noscript>` タグにより「JavaScriptを有効にしてください」と表示される
- [ ] Next.js Hydration 差分によるエラーが発生しない（ブラウザコンソールに `hydration warning / error` が出ないことで確認）

## 8. 機能要件

### 時刻表示
- `setInterval`（1000ms間隔）で時刻を更新する
- タイムゾーン：ブラウザのローカルタイムゾーンに従う（変更不可・v1固定）
- 表示形式：24時間表記、ゼロパディングあり（HH:MM:SS）
- Next.js Hydration 対策：`useEffect` 内で時刻を初期化し、SSR/CSRの差分を防ぐ
- タブ非アクティブ対策：`visibilitychange` イベントを監視し、タブ復帰時に即座に時刻を再取得する
- JS無効対応：`<noscript>` タグに「JavaScriptを有効にしてください」メッセージを設置する

### 日付表示
- `Intl.DateTimeFormat` を使用してja-JPロケール固定で表示
- 表示形式：YYYY年M月D日 曜日（月・日はゼロ埋めなし）
- 曜日の表記：`long`（月曜日・火曜日...）

### UI
- フォント：モノスペースフォントを使用して桁ズレを防止（フォントスタックは `functional-design.md` 参照）
- カラー：時刻テキストと背景のコントラスト比 **4.5:1以上**（WCAG AA準拠）
- アクセシビリティ：時刻要素に `aria-label="現在時刻"` 、日付要素に `aria-label="今日の日付"` を付与し、スクリーンリーダーで意味の通る読み上げ順とする

## 9. 非機能要件

| 項目 | 要件 | 測定条件 |
|------|------|----------|
| パフォーマンス | 時刻文字列の初回表示 1秒以内 | Vercel Preview環境、モバイル設定、Lighthouseデフォルト（Fast 3G相当） |
| Lighthouse | Performance・Accessibility 各90以上 | 本番ビルドのトップページをモバイル設定で計測 |
| アクセシビリティ | WCAG 2.1 AA準拠 | axe・Lighthouse Accessibility で計測 |
| ブラウザ対応 | Chrome・Firefox・Safari・Edge 各最新2バージョン | — |
| レスポンシブ | 320px〜1920px | 代表5ビューポートで目視確認 |
| テスト | ユニットテストカバレッジ 80%以上 | Vitest / Jest の statement・branch・function・line すべてで80%以上 |
